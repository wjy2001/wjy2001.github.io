MySQL（软件系统）总结

软件架构

MySQL是C/S架构的软件

软件的架构有：B/S、C/S

——B/S：B:浏览器，S:服务器（网页上直接能运行，如京东，淘宝）

——C/S：C:客户端，S:服务器（需要安装或下载客户端的软件）

数据库的基本分类

关系型数据库：MySQL、Oracle、DB2……

非关系型数据库：Mongo dB、Redis……

SQL脚本

1、一堆SQL代码，后缀为.sql的文件

DB、DBMS、SQL

DB： Database 数据库

​ 主要是表，表是数据库基本组成单元

DBMS： Database Management System，数据库管理系统

SQL：结构化查询语言，标准的关系型数据库语言

三者的关系：DBMS—执行–>SQL–操作–>DB

表

（1）行：记录/数据，存储一条完整的信息

​ —-数据

​ —-数据类型

​ —-数据约束

（2）列：列名，存储列的相关信息

SQL，操作或控制数据库（DB）

​ —-DQL（数据查询语言）：主要是对SELECT操作的支持

​ —-DML（数据操作语言）：数据库对增（INSERT）删（DELETE）改（UPDATE）

​ —-TCL（事务控制语言）：提交（COMMIT）回滚（ROLLBACK）

​ —-DCL（数据控制语言）：授权（GRANT）等

​ —-DDL（数据定义语言）：创建（CREATE）删除（DROP）修改（ALTER）等支持

SELECT————-查询

FROM—————来自

DISTINCT———-去除重复（行）

FROM子句：确定数据的来源

子句的书写顺序

增删改(DML操作)

增(INSERT)

关键字：INSERT INTO,VALUES

创建数据库

查询表结构

DESC emp(表名)

无条件查询

1.查询表的全部信息

SELECT  *
FROM 表名(表别名);

2.查询表对应列的全部信息

3.无重复的查询表的全部信息

SELECT  DISTINCT *
FROM 表名(表别名);

2.无重复的查询表对应列的全部信息

​ SELECT、FROM、WHERE

子句的执行顺序

FROM……WHERE……SELECT……

基本操作符

​ 基本比较运算符

​ <、>、=、!=(不等于)、<>(不等于)、>=、<=

​ 基本逻辑运算符

​ 能够进行多个条件的连接，这些条件之间构成一定的逻辑关系

​ 与 AND

​ 或 OR

​ 非 NOT

特殊的比较运算符

​ 基本空值判断

​ 判断是NULL的情况：列名：列名(字段名) IS NULL;

错误情况：
WHERE 列名(字段名) = NULL;

​ 判断不是NULL的情况：列名：列名(字段名) IS NOT NULL;

BETWEEN……AND……

在value1到value2之间(包含这两个值，即[value1,value2])

WHERE 列名(字段名) BETWEEN value1 AND value2;

​ 等同于

WHERE 列名(字段名) >= value1 AND 列名(字段名) <= value2;

​ 可以对日期进行比较

模糊查询（LIKE）

​ 关键字like

​ 重要的符号（通配符）：

​ ‘_’：代表一个字符

​ ‘%’：代表的是任意个字符（包含0个）

​ 如查询倒数第二个字为A的信息

WHERE 列名 LIKE '%A_';

注意：这两个字符可以重复使用

IN操作符

数据中不等于value1或value2……

WHERE 列名 NOT IN (value1,value2……)

数据中等于value1或value2……

WHERE 列名 IN (value1,value2……)

数据为NULL时有问题（看子查询）

分页查询（LIMIT）

显示查询结果的前number条数据

SELECT 列名
FROM 表名
LIMIT number;

显示前指定页数的数据，size1是页数，number2是一页中的数据条数

SELECT 列名
FROM 表名
LIMIT size1,number2;

子查询（SELECT嵌套）

​ 子查询顾名思义一个查询（主查询，外部查询）中包含其他查询。

​ 单行子查询（简单子查询）

子查询需要使用( )括起来

单行子查询可以使用基本比较运算符

子查询是可以脱离外部查询独立执行

子查询的书写位置

​ 1.SELECT子句（罕见）

​ 2.FROM子句（可以用，注意别名）

这样写会报错

1248 - Every derived table must have its own alias

错误原因 没有给子查询出来的表起别名

SELECT *
FROM (SELECT * FROM EMP WHERE DEPTNO='20')
where sal>1500;

正确写法

SELECT temp.*
FROM (SELECT * FROM EMP WHERE DEPTNO='20') temp
where temp.sal>1500;

​ 3.WHERE子句（常用）

​ 4.HAVING子句

​ 多行子查询

多行子查询不能使基本比较运算符

能使用

IN

NOT IN

​ 注意事项：( )里如果有NULL，相当于=NULL，无结果(返回值)。

ALL：所有

ANY：任何一个

​ 注意事项：使用ALL或ANY的效率比使用组函数，如MAX()和MIN()更高

多表联合查询

我们需要的数据不可能都来自于一张表中

交叉连接（笛卡尔积）（CROSS JOIM）

笛卡尔积：是指数学中两个集合的乘积

注意：多表查询的时候需要避免笛卡尔积（产生大量冗余数据）

避免方法：一定要写多表连接的连接条件

关键字：CROSS JOIM

等值连接(必须会且熟练)(INNER JOIN……ON)

使用不同表中的具有相同意义的字段(列)相等作为多表的连接条件

方式一：使用WHERE子句添加多表的连接条件

SELECT *
FROM 表1,表2……
WHERE 表1.列名=表2.列名

方式二(通用):使用INNER JOIN……ON(等价于 JOIN……ON)

SELECT *
FROM 表1
INNER JOIN 表2 ON (表1.列名=表2.列名);

注意：1.可以使用表的别名，使用方式和列一样

​ 2.如果不同表中的字段名一致，使用的时候需要区分

不等值连接

等值连接以外的那部分数据

外连接

左外连接:主要显示的是左边表所有行的数据

SELECT *
FROM 表1
LEFT JOIN 表2 ON(表1.列名=表2.列名);

右外连接:主要显示的是右边表所有行的数据

SELECT *
FROM 表1
RIGHT JOIN 表2 ON(表1.列名=表2.列名);

全外连接(MYSQL不支持,但Oracle支持):

SELECT *
FROM 表1
FULL JOIN 表2 ON(表1.列名=表2.列名);

自连接

自己连接自己(表中的A列和B列实际意义相同)

SELECT e.ENAME,e.EMPNO,m.ENAME,m.EMPNO
FROM emp e,emp m
WHERE e.MGR=m.EMPNO

分组查询（GROUP BY）

​ 显然用于进行统计时5个常用函数

关键字group by（按照……分组）

子句：GROUP BY+按照分组的列名（字段名）

书写顺序：

SELECT—>FROM—>WHERE—>GROUP BY—>ORDER BY

注意：

SELECT中可以有GROUP BY的对象

mysql（5.x版本）中执行顺序

FROM—>WHERE—>SELECT—>GROUP BY—>ORDER BY

即分组查询可以使用列别名（只限于目前）

分组条件(HAVING)

​ 关键字：HAVING

​ 子句：HAVING + 分组条件

​ 子句作用：添加分组条件（WHERE语句中不支持分组函数的分组条件）

​ 书写顺序：SELECT—>FROM—>WHERE—>GROUP BY—>ORDER BY

内置函数

​ 函数：在计算机中通常称为方法、函数

​ 函数之间可以嵌套使用

单行函数

​ 处理NULL值

IFNULL(value1, value2)

​ 如果value1为NULL，则返回值为value2

​ 如果value1不为NULL，则返回值为value1

​ 日期类型

​ 获取当前日期：CURDATE()

​ 获取当前时间：CURTIME()

​ 获取当前的日期+时间：NOW()

​ 数值类型

​ 四舍五入：ROUND()

​ 向下取整：FLOOR()

​ 向上取整：CEIL()

面试时注意负数

​ 字符串类型

​ 字符串拼接：CONCAT(string1，string2)–将string1与string2拼接

​ 字符串长度：LENGTAH(string)

​ 去除前后空格：TRIM(string)

​ 转成大写字母：UPPER(string)

​ 转成小写字母：LOWER(string)

多行函数（分组函数）

​ 统计数据总条数：COUNT()

注意：COUNT()括号中为数字或*

面试题：

​ 给你一个陌生的数据库，有一张陌生的表，第一件事你干啥？

（1）SELECT * FROM 表名
（2）SELECT COUNT(*) FORM 表名

​ 选（2）由于这个表的数据量可能会很大，（2）查询数据量更快

​ 统计平均数：AVG()

​ 统计总和：SUM()

​ 统计最大值：MAX()

​ 统计最小值：MIN()
