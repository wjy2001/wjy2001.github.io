ubuntu安装etcd及部署集群

安装etcd

首先去github下载程序包https://github.com/etcd-io/etcd/releases

然后拷贝到ubuntu中

解压

$ tar xzvf etcd-v3.5.4-linux-amd64.tar.gz

进入到对应文件夹，拷贝。

$ cd etcd-v3.5.4-linux-amd64
$ cp etcd etcdctl /usr/local/bin

检验是否成功

$ etcdctl version

部署集群

环境

阿里云服务器一台，

公网ip：101.200.87.232 ，内网ip：172.25.39.58

腾讯云服务器一台，

公网ip：101.43.208.214 ，内网ip：10.0.16.12

设置

静态部署

如果在启动前我们知道集群成员，他们的地址和集群的大小，我们可以使用通过设置 initial-cluster 标记来离线启动配置。

如果使用同样的配置启动多个集群(或者创建并部署单个集群)用于测试目的，强烈推荐每个集群给予一个唯一的 initial-cluster-token。这样做之后，etcd 可以为集群生成唯一的集群 ID 和成员 ID，甚至他们有完全一样的配置。这可以将 etcd 从可能让集群孵化的跨集群交互中保护起来。

etcd 在 listen-client-urls 上接收客户端访问。etcd 成员将 advertise-client-urls 指定的 URl 上通告给其他成员，代理和客户端。常见的错误是设置 advertise-client-urls 为 localhost 或者留空为默认值，如果远程客户端可以达到 etcd。

注意： 在 initial-cluster 中指定的 URL 是 advertised peer URLs ，例如，他们将匹配对应节点的 initial-advertise-peer-urls 的值。

阿里云

$ etcd --name etcd0 --initial-advertise-peer-urls http://101.200.87.232:2380 \
  --listen-peer-urls http://172.25.39.58:2380 \
  --listen-client-urls http://127.0.0.1:2379,http://172.25.39.58:2379 \
  --advertise-client-urls http://127.0.0.1:2379,http://101.200.87.232:2379 \
  --initial-cluster etcd0=http://101.200.87.232:2380,etcd1=http://101.43.208.214:2380 \
  --initial-cluster-state new \
  --initial-cluster-token etcd-cluster

腾讯云

$ etcd --name etcd1 --initial-advertise-peer-urls http://101.43.208.214:2380 \
  --listen-peer-urls http://10.0.16.12:2380 \
  --listen-client-urls http://127.0.0.1:2379,http://10.0.16.12:2379 \
  --advertise-client-urls http://127.0.0.1:2379,http://101.43.208.214:2379 \
  --initial-cluster etcd0=http://101.200.87.232:2380,etcd1=http://101.43.208.214:2380 \
  --initial-cluster-state new \
  --initial-cluster-token etcd-cluster

动态部署

在一些案例中，集群伙伴的 IP 可能无法提前知道。当使用云提供商或者网络使用 DHCP 时比较常见。在这些情况下，相比指定静态配置，可以使用已经存在的 etcd 集群来启动一个新的。我们称这个过程为”发现”。

公共 etcd 发现服务

如果没有现成的集群可用，可以使用托管在 discovery.etcd.io 的公共发现服务。为了使用”new” endpoint来创建私有发现URL，使用命令：

$ curl https://discovery.etcd.io/new?size=2//这里的size是集群的大小，默认是3
https://discovery.etcd.io/160770a95d8bdef7a625b605494eb98b//返回值

阿里云

$ etcd --name etcd0 --initial-advertise-peer-urls http://101.200.87.232:2380 \
  --listen-peer-urls http://172.25.39.58:2380 \
  --listen-client-urls http://127.0.0.1:2379,http://172.25.39.58:2379 \
  --advertise-client-urls http://127.0.0.1:2379,http://101.200.87.232:2379 \
  --discovery https://discovery.etcd.io/160770a95d8bdef7a625b605494eb98b

腾讯云

$ etcd --name etcd1 --initial-advertise-peer-urls http://101.43.208.214:2380 \
  --listen-peer-urls http://10.0.16.12:2380 \
  --listen-client-urls http://127.0.0.1:2379,http://10.0.16.12:2379 \
  --advertise-client-urls http://127.0.0.1:2379,http://101.43.208.214:2379 \
  --discovery https://discovery.etcd.io/160770a95d8bdef7a625b605494eb98b

检测是否成功

$ etcdctl member list

检查集群中的数据是否一致

在阿里云中

$ etcdctl put aaa bbb

在腾讯云中

$ etcdctl get aaa
